### Context
Guest users play with `guest_<uuid>` IDs stored in localStorage only. DB sessions are SKIPPED for guests (use-game-session.ts:38). Migration creates a new DB child under the authenticated parent and updates localStorage via `setChild()`.

### Acceptance Criteria
- T1: Auth via `parentId` cookie; validate body; create Child; return `{ child }`; 400/401/500 handled
- T2: Dismissible; re-shows every session until migrated; calls `onSave` callback; matches design tokens
- T3: Detects guest + parent logged in via `GET /api/auth/session`; shows banner; ParentGate on tap; calls migrate; updates context; clears banner

### Security Constraints (Reviewer must verify)
- No IDOR: migrated child must belong to the authenticated parent
- No double-migration: idempotent if called twice
- Guest data cleared AFTER successful migration only
- No sensitive data in client-visible error messages
