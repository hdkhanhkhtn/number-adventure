# PROMPT – Sitemap / IA / User Flow

```markdown
Create a complete information architecture and user flow for a mobile-first preschool math learning web app called “Bap Number Adventure”.

## Product Summary
A gamified math learning app for a 4-year-old child who already recognizes English numbers and is now progressing into:
- tens, hundreds, thousands
- even and odd
- basic addition and subtraction
- number sense

## Main Users
- child
- parent/caregiver

## Build the Information Architecture
Please structure the app into two major areas:
1. Child Mode
2. Parent Mode

## Child Mode sections
- Splash / welcome
- Child profile
- Home dashboard
- Daily goal card
- World map
- Level/lesson selection
- Mini-game session
- Reward / stars / celebration
- Sticker collection / collectibles
- Progress path
- Resume last activity

## Parent Mode sections
- Parent gate / lock screen
- Child progress overview
- Skills breakdown
- Learning history
- Session time settings
- Language / audio settings
- Difficulty / guided mode settings
- Profile management

## Required User Flows
Please map and design these user flows:
1. First-time onboarding flow
2. Returning child quick play flow
3. Select world -> select lesson -> play mini-game -> reward -> next lesson
4. Daily 5-minute learning session
5. Parent enters protected mode
6. Parent reviews progress and adjusts settings
7. Child earns stickers and unlocks new content
8. Locked lesson / lesson completion flow

## Navigation Model
Recommend the most suitable navigation model for mobile-first use.
Prioritize:
- simplicity
- low friction
- intuitive child use
- parent-safe separation

## Output Needed
Please produce:
- sitemap
- information architecture
- navigation structure
- screen hierarchy
- user flow diagrams in words
- notes on what should be persistent navigation vs contextual navigation
- UX reasoning for preschool child behavior
```
