# PROMPT – Mini-game 4: Even / Odd Sorting

```markdown
Design the UI for a mini-game called “Even or Odd House” in the app “Bap Number Adventure”.

## Learning Goal
Introduce the idea of even and odd numbers in a visual, playful way.

## Core Concept
The child sorts numbers into:
- Even House
- Odd House

The concept should first feel intuitive and visual, not abstract.

## Interaction
- number cards appear one by one or in a group
- child drags card into Even House or Odd House
- optional visual pairing support

## Two Modes
### Mode 1: Visual Pairing Intro
Use objects (e.g. apples, stars, socks) to show:
- even means objects can be paired with none left over
- odd means one is left over

### Mode 2: Number Sorting
Child sorts written numbers into even or odd categories.

## UI Requirements
- two large destination areas
- very clear color distinction
- large draggable number cards
- optional pairing animation
- friendly mascot explanation
- low cognitive load

## Show These States
- intro teaching state
- drag state
- correct drop
- incorrect drop
- visual explanation state
- round complete state

## Theme Suggestions
- two cute houses
- bridge with two gates
- baskets or doors
Use a delightful metaphor appropriate for preschool learners.

## Output
Design the full mobile UI flow for both the teaching phase and sorting game phase.
```
