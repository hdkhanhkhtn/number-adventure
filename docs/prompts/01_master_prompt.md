# MASTER PROMPT – Thiết kế toàn bộ MVP

```markdown
Design a complete mobile-first UI/UX for an MVP educational web app for preschool kids (around 4 years old) called “Bap Number Adventure”.

## Product Goal
The app helps a young child gradually build early math thinking through gamified mini-games. The child already recognizes and reads numbers in English from 1 to 100, including hundreds and thousands at a basic recognition level. The next learning goals are:
- become more fluent with tens, hundreds, thousands, and larger numbers
- begin learning even and odd numbers
- begin learning basic arithmetic
- strengthen number sense through playful interaction

## Target Users
### Primary user
- Preschool child around age 4
- Limited reading ability
- Can listen to English prompts and choose the correct number
- Learns best through visuals, sound, repetition, touch interactions, and short activities

### Secondary user
- Parent or caregiver
- Wants to track progress, choose learning mode, and manage screen time

## Platform
- Mobile-first web app
- Optimized for smartphone and tablet
- PWA-friendly
- Touch-first interaction
- Responsive but prioritize mobile layouts first

## Product Style
- Warm, playful, safe, colorful
- Cute but not cluttered
- Friendly mascot
- Soft rounded shapes
- Big buttons
- Large touch targets
- Clear visual hierarchy
- Child-friendly illustrations
- Rewarding animations
- Delightful but calm, not overstimulating

## Core UX Principles
- One primary task per screen
- Very low cognitive load
- Very large buttons and tap areas
- Minimal text on kid-facing screens
- Audio-first guidance
- Error recovery should feel encouraging, never punishing
- Immediate feedback
- Very short game loops (30 seconds to 2 minutes)
- Parent area should be separate and protected

## Brand / Theme
Create a whimsical math adventure world with a cute mascot companion.
Possible mood: magical learning garden + playful number world.
Use a visual system that can scale across multiple worlds:
- Number Forest
- Tens Town
- Big Number Castle
- Even/Odd Bridge
- Math Kitchen

## MVP Features
Design the full UI for these key modules:
1. Onboarding / profile creation
2. Child home screen
3. World map / level selection
4. Daily play session
5. Mini-game UI templates
6. Reward / celebration screens
7. Collection / stickers / simple progression
8. Parent dashboard
9. Settings / language / audio options

## Mini-games included in MVP
1. Hear and Tap Number
2. Number Order / Missing Number
3. Build the Number (tens and ones)
4. Even / Odd Sorting
5. Basic Add / Take Away (intro level)

## Required Screens
Please create a full UI flow including:
- Splash screen
- Welcome screen
- Child profile setup
- Parent gate
- Home dashboard
- World map
- Level list / lesson node screen
- Mini-game screen templates
- End of round feedback
- Reward screen
- Daily progress / streak screen
- Sticker or collectible screen
- Parent dashboard home
- Parent progress report
- Parent settings
- Session time limit settings
- Audio/language settings

## Design System Requirements
Create a reusable component system with:
- typography scale
- color palette
- icon style
- rounded cards
- buttons
- progress bars
- audio buttons
- answer tiles
- drag and drop slots
- character / mascot placement guidelines
- reward badges / star indicators

## Accessibility Requirements
- very high legibility
- simple shapes
- high contrast where needed
- consistent placement of key controls
- visual + audio cues
- avoid text-heavy layouts
- touch-friendly spacing
- support for children with early learning stage

## Output Required
Please generate:
1. visual design direction
2. information architecture
3. mobile screen designs for all key screens
4. a reusable design system
5. core components
6. interaction notes
7. microcopy examples
8. responsive notes for tablet
9. empty, loading, success, error states
10. parent mode screens

## Important Content Notes
The app should be bilingual-friendly:
- kid-facing interactions can use English audio prompts
- parent dashboard can use Vietnamese or English-friendly copy
- include easy toggles for English only / Vietnamese only / bilingual mode

## Emotional Design
The child should feel:
- safe
- successful
- curious
- excited to continue
- proud of progress

The parent should feel:
- informed
- in control
- reassured
- confident the child is learning meaningfully

## Deliverable Style
Provide the UI in a polished, production-ready style suitable for MVP prototyping and developer handoff.
```
