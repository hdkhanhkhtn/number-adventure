# PROMPT – Developer Handoff

```markdown
Create a UI handoff specification for the MVP of “Bap Number Adventure”, a mobile-first preschool math learning web app.

## Purpose
The output should help a development team implement the MVP efficiently.

## Include
### 1. Screen Inventory
List all screens and variants in the MVP.

### 2. Components
Document all reusable components:
- buttons
- cards
- progress bars
- number tiles
- answer options
- drag items
- drop zones
- reward modals
- bottom navigation
- parent stat cards

### 3. Layout System
- mobile breakpoints
- spacing scale
- safe touch target sizes
- responsive rules for tablet

### 4. Design Tokens
- colors
- typography
- radius
- spacing
- shadows
- icon sizing

### 5. States
- default
- hover if web
- pressed
- disabled
- selected
- correct
- incorrect
- loading
- empty

### 6. Interaction Notes
- audio replay behavior
- correct/incorrect feedback timing
- drag-and-drop behavior
- reward modal behavior
- parent gate behavior

### 7. Content Notes
- child-facing copy
- parent-facing copy
- bilingual support
- dynamic content examples

### 8. Accessibility Notes
- touch areas
- contrast
- audio alternatives
- consistency of control placement

## Output
Provide a clean, developer-friendly, implementation-ready design handoff structure.
```
