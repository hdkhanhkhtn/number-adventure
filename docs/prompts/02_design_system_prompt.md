# PROMPT – Design System

```markdown
Create a full design system for a mobile-first educational web app for preschool children called “Bap Number Adventure”.

## Brand Personality
- playful
- warm
- safe
- intelligent
- encouraging
- magical
- child-friendly

## Target Users
- young children around age 4
- parents/caregivers

## Design Goals
The design system should support a gamified early math app with many mini-games and learning worlds. It should feel delightful but not chaotic. It should be scalable across lessons, game types, rewards, and parent dashboard screens.

## Create the Following
### 1. Color Palette
Define:
- primary brand colors
- secondary colors
- accent colors
- feedback colors (success, warning, error, neutral)
- background colors
- card colors
- disabled states
Use soft, cheerful, child-safe colors with good readability.

### 2. Typography
Define a typography system for:
- large playful titles
- section headings
- button labels
- game instructions
- score labels
- parent dashboard text
Typography should be rounded, highly legible, and easy to read on mobile.

### 3. Iconography
Create an icon direction:
- rounded
- simple
- friendly
- easy for children to understand
Key icons: home, audio, reward, world map, profile, parent mode, settings, next, back, lock, star, sticker, progress.

### 4. Components
Design reusable components for:
- primary button
- secondary button
- icon button
- audio replay button
- progress bar
- reward badge
- bottom navigation
- card
- lesson tile
- answer option tile
- draggable item
- drop zone
- number card
- mascot speech bubble
- modal / dialog
- celebration banner
- daily streak card
- sticker card
- parent dashboard stat card

### 5. UI Patterns
Design patterns for:
- game instruction header
- single-question game screen
- drag-and-drop interaction
- multiple choice interaction
- quick success feedback
- gentle retry feedback
- end-of-session summary
- parent-only gated section

### 6. Character / Mascot Usage
Define how the mascot appears:
- onboarding
- encouragement
- success celebration
- hints
- reward screens
Ensure mascot enhances engagement but does not distract from learning tasks.

### 7. Layout Rules
Define spacing, margins, corner radius, elevation, and safe touch target size.
All layouts should be optimized for one-hand or two-hand mobile/tablet child interaction.

### 8. States
Design states for:
- default
- pressed
- active
- selected
- correct
- incorrect
- disabled
- locked
- completed

## Deliverables
Please produce:
- a coherent design system board
- token-like definitions
- component set
- visual usage examples on screens
- mobile-first specifications
- tablet adaptation notes
```
