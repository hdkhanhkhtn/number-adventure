# PROMPT SIÊU TỔNG HỢP – All-in-one

```markdown
Design a complete mobile-first MVP UI/UX for a gamified preschool math learning web app called “Bap Number Adventure”.

## Product Overview
This app is for a child around 4 years old who already recognizes and reads numbers in English from 1 to 100, and has basic awareness of hundreds and thousands. The app should help the child gradually become more fluent with:
- tens, hundreds, thousands, and larger numbers
- even and odd numbers
- number structure
- number order and missing numbers
- early addition and subtraction
- number sense through playful interaction

The app should be optimized for smartphone and tablet, mobile-first, touch-first, and suitable as a web app / PWA.

## Users
### Primary user
A preschool child around 4 years old with limited reading ability but able to listen to English prompts and choose the correct number.

### Secondary user
A parent or caregiver who manages settings and tracks progress.

## UX Principles
- one primary task per screen
- very large touch targets
- low cognitive load
- audio-first guidance
- minimal text on child-facing screens
- visually rewarding
- encouraging feedback
- no punitive feeling
- short game loops
- safe and calm emotional experience

## Product Areas
Design the full UI for:
1. onboarding
2. child home
3. world map
4. level selection
5. mini-game templates
6. Hear and Tap Number game
7. Missing Number / Number Order game
8. Build the Number game
9. Even or Odd Sorting game
10. Basic Add / Take Away game
11. reward and celebration screens
12. sticker / collection screens
13. daily progress / streak screen
14. parent gate
15. parent dashboard
16. parent progress details
17. parent settings
18. all key states: loading, empty, locked, completed, error, success

## Worlds
Use a whimsical adventure structure with worlds such as:
- Number Forest
- Tens Town
- Big Number Castle
- Even/Odd Bridge
- Math Kitchen

## Visual Style
- warm
- playful
- magical
- safe
- rounded
- soft colorful palette
- cute mascot companion
- clean layout
- polished modern educational product quality

## Design System
Create a reusable design system including:
- color palette
- typography
- buttons
- cards
- answer tiles
- progress bars
- reward components
- drag and drop components
- iconography
- mascot usage guidelines

## Child Home Requirements
Include:
- greeting
- mascot
- Play Now CTA
- daily goal
- current world
- world map shortcut
- stars earned
- sticker shortcut
- simple progress indicators

## Game Screen Requirements
Use a reusable layout with:
- progress bar
- audio replay button
- mascot hint area
- main prompt area
- large answer area
- feedback animations
Support multiple choice, drag-and-drop, order sequence, and visual arithmetic patterns.

## Reward Requirements
Design:
- correct answer feedback
- mini celebration
- end-of-session summary
- sticker unlock
- world unlock
- achievement milestone

## Parent Mode
Design:
- parent gate
- parent dashboard overview
- progress details
- settings
- language mode
- audio settings
- session time settings
- profile settings

## Content Direction
Support language mode:
- English only
- Vietnamese only
- Bilingual

Child-facing screens should rely more on visuals and audio than text.
Parent-facing screens should be cleaner and more informational.

## Accessibility
- large type
- high legibility
- consistent controls
- touch-friendly spacing
- supportive visual and audio cues

## Output
Please generate:
1. full information architecture
2. screen inventory
3. complete mobile UI screens
4. component-based design system
5. state variants
6. interaction notes
7. tablet adaptation notes
8. prototype flow suggestions
9. developer handoff readiness

The result should be polished, MVP-ready, and suitable for design-to-development workflow.
```
