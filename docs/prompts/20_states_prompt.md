# PROMPT – Empty / Loading / Error / Success States

```markdown
Design all important UI states for the mobile-first preschool math app “Bap Number Adventure”.

## States Needed
1. Loading state
2. Empty state
3. No progress yet state
4. Locked lesson state
5. Completed lesson state
6. Temporary offline or error state
7. Retry state
8. Save success state
9. Session complete state

## Style Requirements
- consistent with brand
- warm and non-threatening
- supportive
- child-friendly for child mode
- clearer and more informative for parent mode

## Examples
### Loading
- animated mascot
- fun loading phrase

### Empty progress
- encouraging start message
- CTA to begin first lesson

### Locked lesson
- visual lock
- explain what needs to be completed first

### Error
- gentle message
- clear retry button
- avoid technical jargon

### Success
- celebratory feedback
- very clear confirmation

## Output
Design visual state screens or state variants for key components and pages.
```
