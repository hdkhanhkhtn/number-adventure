Bắp Number Adventure

Complete IA, screen hierarchy, navigation model & 8 user flow diagrams for a mobile-first preschool math app.

🌱 Child Mode 👤 Parent Mode 📱 Mobile-first 🎯 Age 4 🌍 EN / VI bilingual
🎯 Product Goal

Help a 4-year-old child build early math fluency through short, gamified mini-game sessions, supported by a parent dashboard for progress tracking and controls.

📐 IA Principles
  • One primary task per screen
  • Child & parent areas strictly separated
  • Minimal depth for child paths (≤3 taps to any game)
  • Audio-first, text-minimal child UI
  • Parent gate blocks all parent screens

Navigation Model

Recommended: Hub & Spoke

The recommended model is Hub & Spoke with contextual back arrows. Home is the single hub. All child destinations are spokes. There is no persistent bottom tab bar for kids — it adds cognitive load and accidental taps for a 4-year-old. Navigation is done through large illustrated buttons on the home screen instead.

Why not tabs for kids? Tabs require reading labels and making abstract categorical choices. A 4-year-old navigates better through a single illustrated home screen with large, colourful picture buttons — the app should feel like one flowing world, not a tabbed interface.

UX Reasoning — Preschool Child Behavior

👁️
Visual memory over reading
A 4-year-old navigates by pictures, color, and position — not by reading labels. Every navigation element must have a distinct visual identity. Never rely on text alone.
👆
Big, forgiving touch targets
Minimum 64px tap targets for primary actions. Key buttons (Play, number tiles) should be 80–140px. Children have imprecise motor control — leave generous spacing between tappables.
⏱️
Micro-loops under 30 seconds
Each question round resolves in ≤30 seconds. Each game session is 2–3 minutes maximum. Children this age have short working memory and need rapid feedback loops to stay motivated.
🔊
Audio-first, text-minimal
All instructions are delivered by audio (speech synthesis or recorded). Text on kid screens is for number recognition only — the one thing Bắp can read. No button labels needed if icons are clear.
🌈
Color is navigation
Each world has a unique color identity. Each feedback state (correct/wrong) has consistent color. Children learn to associate green=good, warm orange=try again — use these consistently.
💛
Never punish — always encourage
Wrong answers cause a gentle wiggle animation and lose one heart. The mascot reacts with encouragement, never disappointment. The retry experience feels like a new chance, not a failure.
🎁
Variable reward keeps engagement
Stickers appear randomly on 3-star completions (not guaranteed). This variable reward schedule is highly motivating for children without being manipulative — the child always earns coins and stars.
🗺️
Spatial navigation (world map)
Young children understand space and place better than abstract categories. The world map gives them a mental model of "I am here, I want to go there" — far more intuitive than a content library grid.
🐣
Mascot as safety anchor
Bắp the mascot provides a consistent emotional anchor. Seeing the same friendly character react to what happens makes the experience feel personal and safe — especially important for a child using an app independently.
📵
No external links or notifications
The app should feel like a closed, safe world. No external links, no push notifications without parental opt-in, no social features. Parents should always feel fully in control.
↩️
Persist state across sessions
Always remember where the child left off. Session state, progress, and stickers persist via localStorage. Seeing your progress exactly where you left it is reassuring and reduces re-orientation time.
🔒
Parent gate is non-negotiable
The math problem gate must be unsolvable by a typical 4-year-old. Use sums in the 5–15 range with 4 decoy answers. Never show a PIN — parents forget them. The gate protects both the settings and the child's mental model of a "safe play space."
