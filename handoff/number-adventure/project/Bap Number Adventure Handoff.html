Bắp Number Adventure

Mobile-first preschool math web app (ages 3–6). Audio-led gameplay, bilingual support (EN/VI), parent dashboard in Vietnamese. All screens are single-column, touch-first, 390 px reference width.

React · Hooks Mobile-first Fredoka + Be Vietnam Pro Web Speech API CSS custom properties

Section 06

Interaction Notes

Timing and sequencing for all key micro-interactions and flows. All durations are in milliseconds unless otherwise noted.

Audio Replay (Hear & Tap)

EventBehaviour
Round mountAuto-play via Web Speech API (speechSynthesis.speak()). Rate: 0.85, pitch: 1.1, lang: en-US
Speaker button tapCancel any in-progress speech → replay same number
API unavailableSilently no-op (if (!('speechSynthesis' in window)) return) — game still fully playable visually
Bilingual mode (future)Play English first, then Vietnamese after 0.8 s pause
Speaker button uses pulse-soft animation on idle, stops pulsing after first tap. Do not replay audio automatically between rounds — always wait for explicit user tap.

Correct Answer Feedback

TimeAction
0 msSet tile state → correct. Tile turns green, translateY(-2px)
0 msSet correctPicked = true — all other tiles become non-interactive
900 msAdvance to next round (or call onRoundEnd if last round)

Wrong Answer Feedback

TimeAction
0 msSet tile state → wrong. Tile turns coral + wiggle animation (0.4 s)
0 msDecrement hearts (Math.max(0, h - 1))
0 msAdd to wrongs Set — tile stays visually wrong, but re-tap is silently blocked
—Correct tile remains hittable. No auto-advance; user must find correct answer
Hearts floor at 0 — never show negative. At 0 hearts, game completes with 1 star regardless of remaining questions.

Button Press Physics

// onMouseDown (and onTouchStart equivalent)
style.transform = 'translateY(4px)';
style.boxShadow = `0 1px 0 ${c.sh}, 0 3px 8px …`;

// onMouseUp + onMouseLeave (reset)
style.transform = '';
style.boxShadow = `0 5px 0 ${c.sh}, 0 10px 20px …`;

// CSS transition on all interactive buttons:
transition: 'transform 0.1s ease, box-shadow 0.1s ease';

Drag-and-Drop Behaviour (Build Number)

The BuildNumber game uses + / − tap buttons (not drag) for the MVP. Tens and ones columns each have their own counter. Visual feedback: each added block uses pop-in animation class.

Drag-and-drop is deferred post-MVP. The current tap-to-add/remove mechanic works reliably on touch without gesture conflicts.

Splash Auto-Advance

setTimeout(onReady, 2200) with cleanup on unmount. If user taps during splash, nothing happens — no skip. Clean up the timer properly to avoid navigation side-effects when the component unmounts.

Session Completion

Each game calls onRoundEnd({ stars, correct, total }) when round + 1 >= rounds. The parent computes a sticker reward: if stars === 3 && Math.random() > 0.5 → random sticker granted. This probability is intentional — not every 3-star run gives a sticker.

Motion & Cognitive Load

All animations are purposeful, not decorative loops during gameplay. bobble and sparkle animations are idle-only — they do not compete with active interaction areas. For children with motion sensitivity, future iterations should honour prefers-reduced-motion:

@media (prefers-reduced-motion: reduce) {
  .bobble, .sparkle, .pulse-soft { animation: none; }
  .pop-in { animation-duration: 0.01ms; }
}
